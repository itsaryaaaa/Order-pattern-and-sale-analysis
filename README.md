# 📊 Retail Sales & Order Pattern Analysis

## 📌 Overview

This project focuses on analyzing retail sales and customer order patterns using Python and PostgreSQL. The objective of the project is to extract meaningful business insights from raw retail transaction data through data cleaning, exploratory data analysis (EDA), and SQL-based analysis.

The project demonstrates an end-to-end analytics workflow including:

* Data preprocessing and cleaning
* Exploratory Data Analysis (EDA)
* SQL-based business analysis using PostgreSQL
* Trend and pattern identification
* Business insight generation

This project was developed to strengthen practical skills in Data Analytics, SQL, and Business Intelligence.

---

# 📂 Dataset Information

The dataset contains retail order transaction data with fields such as:

* Order ID
* Customer Details
* Product Category & Sub-Category
* Sales & Profit
* Quantity Ordered
* Region & State
* Shipping Mode
* Order Date & Ship Date

The dataset was analyzed to identify customer purchasing behavior, sales performance trends, and regional business insights.

# 🔄 Project Workflow

## 1️⃣ Data Loading

* Imported the retail dataset into Python using Pandas
* Inspected dataset structure, columns, and data types

## 2️⃣ Data Cleaning & Preprocessing

Performed:

* Missing value handling
* Duplicate removal
* Date formatting
* Data type conversion
* Basic data consistency checks

## 3️⃣ Exploratory Data Analysis (EDA)

Analyzed:

* Sales trends
* Profit distribution
* Regional sales performance
* Product category analysis
* Customer ordering patterns
* High-performing products

Visualizations were created using Matplotlib and Seaborn for better understanding of trends and insights.

## 4️⃣ SQL Analysis using PostgreSQL

The cleaned dataset was loaded into PostgreSQL for advanced business analysis.

SQL queries were used to analyze:

* Revenue generation
* Top-selling products
* Monthly sales trends
* Customer purchase behavior
* Profitability analysis
* Order patterns across regions
